# 08 チーム体制

## 5人チーム想定の役割

### 1. PM / 進行管理
- 全体進行
- スプリント管理
- タスク整理
- 優先順位調整

### 2. UI / UX 担当
- 画面設計
- デザイン
- Figma作成
- コンポーネント方針整理

### 3. フロントエンド担当
- 画面実装
- 状態管理
- コンポーネント化
- レスポンシブ対応

### 4. バックエンド / DB担当
- 認証
- DB設計
- API設計
- 保存処理

### 5. QA / 公開 / 運用担当
- テスト観点整理
- バグ管理
- リリース手順整理
- 運用ログ確認

## 補足
完全な分業ではなく、各担当が主担当を持ちながら相互レビューする体制を取る。
