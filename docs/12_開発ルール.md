# 12 開発ルール

## ブランチ運用
- main: 本番相当
- develop: 開発統合
- feature/機能名: 個別機能開発
- fix/内容: バグ修正

## コミットメッセージ例
- feat: 企業一覧画面を追加
- fix: ログイン後の遷移不具合を修正
- docs: 企画概要を更新
- refactor: タスク一覧の状態管理を整理

## Pull Requestルール
- 何をしたかを書く
- 動作確認内容を書く
- スクリーンショットがあれば添付する
- 大きすぎる変更は分割する

## Issueルール
- 目的
- 背景
- 完了条件
- 優先度
- 担当者

## チーム開発で大事にすること
- 仕様変更は口頭だけで終わらせない
- docs に残す
- 困った時は早めに相談する
- 完成度よりまず動くものを作る
- 使う人の目線を忘れない
