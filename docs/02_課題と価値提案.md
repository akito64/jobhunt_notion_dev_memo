# 02 課題と価値提案

## 解決したい課題

### 1. 情報が分散している
就活に必要な情報が複数の場所に散らばり、必要な時にすぐ参照しにくい。

### 2. 締切と準備の管理が難しい
ES提出、説明会、面接、Webテストなどが重なり、抜け漏れが起きやすい。

### 3. 振り返りが活かしにくい
面接の反省や改善点を記録しても、次回に再利用しづらい。

### 4. 汎用ツールでは就活向けに整っていない
Notionやメモアプリは自由度が高い一方、就活に必要な構造を自分で作る必要がある。

## 提供価値

### 一元管理
企業研究、ES、面接、タスク、予定を1つのサービス内で管理できる。

### 行動に直結するUI
一覧、カンバン、カレンダー、テンプレートなど、就活行動に直結する見せ方を標準搭載する。

### 改善を積み上げられる
面接ログやES履歴を蓄積し、次回の対策や改善につなげやすくする。

### 就活特化の初期設計
ユーザーが最初から迷わず使えるよう、就活に必要なページ構成やテンプレートをあらかじめ用意する。

## 差別化ポイント
- 汎用メモではなく就活特化
- カンバンとカレンダーを中心に進捗を見える化
- 企業ごとの情報集約がしやすい
- ESや面接ログまでつながる
- 将来的にAI添削や改善提案に拡張しやすい
